Steve Howell
10:58 PM

@Adarsh S The main result of adding avatar_version will be that avatar updates will work even when browsers cache image files. When a user uploads an avatar, they update the bits in their avatar, but the location of the file and the URL for the file remains the same, so browsers may continue to show the old avatar.

If we add something like &version=2 to the url, the browser will use the new avatar.

I think the font used, and especially at this size is quite a step backward

Different fonts differ in 'apparent size' though their pixel size might not. This one is definitely smaller than the one used before

Recently read a long and quite complicated article on font-sizes and usability. For web - optimal is 16-18px (depending on font)

Also these custom 'indy' fonts tend to not have wide language support. This is the case for this font too.

Also, the font alternatives are not good, obviously no one is using this on Windows :D

"Helvetica Neue",Helvetica,Arial,sans-serif => means 'Arial' for Windows which is pretty ugly. What you want is 'Segoe UI' which is the UI font that equals Helvetica Neue in modernity

And last but not least - the font of 'Home', 'Private Messages' is still Helvetica Neue, weight 300 (which is not a weight to use for UI stuff, it feels very iOS 7 where Apple went overboard with thin fonts).
